PRECINCT SDKs

First-class SDK support in Python and Go for integrating agents with the PRECINCT gateway. Both SDKs handle SPIFFE identity injection, structured error mapping, and gateway communication automatically.

First Time Integrating?

The Integration Guide walks through identity registration, tool setup, and policy configuration before you start using the SDK.

Overview

PRECINCT provides two official SDKs that mirror each other's semantics across languages. Both SDKs wrap the gateway's JSON-RPC API and provide idiomatic interfaces for tool invocation, error handling, and identity management.

Python SDK

mcp_gateway_sdk: an async-first Python client with Pydantic models, automatic retry, and structured error types.

View on GitHub →

Go SDK

mcpgateway: an idiomatic Go client with typed responses, context-based cancellation, and zero external dependencies beyond the standard library.

View on GitHub →

Roadmap: Rust, TypeScript, and Elixir SDKs are planned. Follow the GitHub repository for updates.

Python SDK: mcp_gateway_sdk

The Python SDK provides a GatewayClient class that handles all communication with the PRECINCT gateway. It automatically injects the caller's SPIFFE ID into every request and maps gateway error codes to typed Python exceptions.

Basic Usage

from mcp_gateway_sdk import GatewayClient, GatewayError

# Initialize the client with gateway URL and SPIFFE identity
client = GatewayClient(
    gateway_url="http://localhost:9090",
    spiffe_id="spiffe://poc.local/agents/mcp-client/dspy-researcher/dev"
)

# List available tools
tools = await client.list_tools()
for tool in tools:
    print(f"  {tool.name}: {tool.description}")

# Call a tool
result = await client.call_tool(
    name="tavily_search",
    arguments={"query": "AI security best practices"}
)
print(result.content)

Error Handling

from mcp_gateway_sdk import (
    GatewayClient,
    AuthzDeniedError,
    DLPViolationError,
    RateLimitError,
    ToolNotFoundError,
)

try:
    result = await client.call_tool("sensitive_tool", {"data": payload})
except AuthzDeniedError as e:
    # OPA policy denied the request (layer 6)
    print(f"Policy denied: {e.decision_id}")
except DLPViolationError as e:
    # DLP scanner detected credentials (layer 7)
    print(f"DLP blocked: {e.pattern_matched}")
except RateLimitError as e:
    # Rate limit exceeded (layer 11)
    print(f"Rate limited, retry after: {e.retry_after}s")
except ToolNotFoundError as e:
    # Tool not registered (layer 5)
    print(f"Unknown tool: {e.tool_name}")

Key Features

  • Automatic SPIFFE ID injection: every request includes the X-SPIFFE-ID header.
  • Typed error hierarchy: gateway deny codes map to specific Python exception classes.
  • Pydantic models: request and response payloads are validated with Pydantic.
  • Async-first: built on httpx with native async/await support.
  • Structured logging: all gateway interactions are logged in structured JSON format.

Go SDK: mcpgateway

The Go SDK mirrors the Python SDK's semantics with idiomatic Go conventions. It uses standard context.Context for cancellation, returns typed errors, and has zero dependencies beyond the Go standard library.

Basic Usage

package main

import (
    "context"
    "fmt"
    "log"

    "github.com/RamXX/agentic_reference_architecture/POC/sdk/go/mcpgateway"
)

func main() {
    client, err := mcpgateway.NewClient(
        "http://localhost:9090",
        mcpgateway.WithSPIFFEID("spiffe://poc.local/agents/mcp-client/researcher/dev"),
    )
    if err != nil {
        log.Fatal(err)
    }

    ctx := context.Background()

    // List available tools
    tools, err := client.ListTools(ctx)
    if err != nil {
        log.Fatal(err)
    }
    for _, t := range tools {
        fmt.Printf("  %s: %s\n", t.Name, t.Description)
    }

    // Call a tool
    result, err := client.CallTool(ctx, "tavily_search", map[string]any{
        "query": "AI security best practices",
    })
    if err != nil {
        log.Fatal(err)
    }
    fmt.Println(result.Content)
}

Error Handling

result, err := client.CallTool(ctx, "sensitive_tool", args)
if err != nil {
    var authzErr *mcpgateway.AuthzDeniedError
    var dlpErr   *mcpgateway.DLPViolationError
    var rateErr  *mcpgateway.RateLimitError

    switch {
    case errors.As(err, &authzErr):
        // OPA policy denied (layer 6)
        log.Printf("Policy denied: decision=%s", authzErr.DecisionID)
    case errors.As(err, &dlpErr):
        // DLP blocked credentials (layer 7)
        log.Printf("DLP violation: pattern=%s", dlpErr.Pattern)
    case errors.As(err, &rateErr):
        // Rate limit exceeded (layer 11)
        log.Printf("Rate limited, retry after %v", rateErr.RetryAfter)
    default:
        log.Printf("Gateway error: %v", err)
    }
}

Key Features

  • Zero external dependencies: uses only the Go standard library.
  • Context-based cancellation: all methods accept context.Context.
  • Typed error values: gateway deny codes map to Go error types compatible with errors.As.
  • Functional options pattern: client configuration via With* option functions.
  • Connection pooling: reuses HTTP connections via the default transport.

Error Code Mapping

Both SDKs map the gateway's structured deny codes to language-specific error types. This table shows the correspondence.

SDK error type mapping for gateway deny codes
Gateway Code Layer Python Exception Go Error Type
spiffe_auth_required 3 AuthenticationError *AuthenticationError
tool_not_in_registry 5 ToolNotFoundError *ToolNotFoundError
tool_hash_mismatch 5 ToolIntegrityError *ToolIntegrityError
authz_policy_denied 6 AuthzDeniedError *AuthzDeniedError
dlp_credentials_detected 7 DLPViolationError *DLPViolationError
stepup_required 9 StepUpRequiredError *StepUpRequiredError
rate_limit_exceeded 11 RateLimitError *RateLimitError
circuit_breaker_open 12 CircuitBreakerError *CircuitBreakerError

Automatic SPIFFE Identity

Both SDKs can automatically obtain SPIFFE identity from the local SPIRE agent when running inside a PRECINCT-managed environment. This eliminates the need to manually specify a SPIFFE ID.

Python Auto-discovery

# When running inside PRECINCT, the SDK discovers the SVID automatically
# via the Workload API Unix Domain Socket
client = GatewayClient(
    gateway_url="http://localhost:9090",
    # No spiffe_id needed -- discovered from SPIRE agent
)

Go Auto-discovery

// Auto-discover SPIFFE identity from local SPIRE agent
client, err := mcpgateway.NewClient(
    "http://localhost:9090",
    mcpgateway.WithAutoSPIFFE(),  // Connects to SPIRE agent UDS
)
Development Mode

In development (Docker Compose with make up), you can manually set the SPIFFE ID via the constructor parameter. In production (Kubernetes), use auto-discovery to obtain identity from the local SPIRE agent.

Installation

Python

# From the repository
cd POC/sdk/python
pip install -e .

# Or install directly
pip install mcp-gateway-sdk

Go

# Add to your Go module
go get github.com/RamXX/agentic_reference_architecture/POC/sdk/go/mcpgateway
Version Note

Both SDKs are part of the PRECINCT proof-of-concept and are versioned alongside the main repository. For production use, pin to a specific commit or tag.
